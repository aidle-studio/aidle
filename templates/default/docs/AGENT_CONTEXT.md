## プロジェクト方針（要点）
- 本プロジェクトは AI駆動開発 (AIDD) を前提とし、仕様駆動（SDD）とテスト駆動（TDD）を統合して進める。
- 実装前に、対象仕様は必ず合意済みであること。
- 仕様は `REQ-*` / `AC-*`、テストは `TC-*` で追跡する。

## 優先ドキュメント読了順
AIエージェントは以下の順序でドキュメントを読み込み、プロジェクトの全体像を把握すること。

1. `AGENTS.md` (最上位のルール)
2. `docs/RULES.md` (開発ルール)
3. `docs/SPEC.md` (仕様書)
4. `docs/TEST_PLAN.md` (テスト計画)
5. `docs/TODO.md` (タスクと進捗)
6. `docs/KNOWLEDGE.md` (技術的判断の記録)

## 実行ルール
- 変更時は必ず対応するテストを先に `Red` 状態で追加すること（TDDの強制）。
- 実装は最小変更で `Green` にし、その後 `Refactor` すること。
