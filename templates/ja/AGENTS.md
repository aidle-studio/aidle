# AGENTS.md

このプロジェクトは **エージェントファースト（Agent-First）** な開発環境です。
AIエージェント（Gemini, Claude, Cursor等）は、このファイルを起点にプロジェクトの文脈と掟を理解してください。

## 🗺️ プロジェクトの地図 (Core Context)

詳細なルールやコンテキストは `docs/` 配下に配置されています。
エージェントは、参画時に必ず以下の4つのコアファイルを順番に読み込み、前提知識としてください。

1. **[ARCHITECTURE.md](ARCHITECTURE.md)**: プロジェクトの全体構造、依存関係、不変の地図。
2. **[docs/HARNESS.md](docs/HARNESS.md)**: 【重要】セッション開始時に必ず確認し、自分のための検証レールを設営してください。
3. **[docs/AGENT_CONTEXT.md](docs/AGENT_CONTEXT.md)**: 現在のフェーズ、直近ゴール、次アクションの動的コンテキスト。
4. **[docs/RULES.md](docs/RULES.md)**: 開発プロセスと完了条件（DoD）の定義（宪法）。

## 🚀 タスクの進め方

本プロジェクトは **「計画 ➔ ハーネス ➔ 実装 ➔ 監査 ➔ 検証」** のサイクルで進行します。
詳細は `docs/RULES.md` に記載されています。

- **指示の起点**: エージェントが「何をすべきか」迷った際は、常に `docs/AGENT_CONTEXT.md` に戻り、現在のステータスとアクティブなプラン（`docs/exec-plans/active/*.md`）を確認してください。
- **検証優先**: 自然言語の指示よりも、機械的な検証（Lint, Test）を優先してください。
- **継続的プロセス改善**: 現状のルールやドキュメントに不備や改善点を見つけた場合は、積極的に改善案を提案してください。

## 🤖 初期セットアップ

新しくプロジェクトに参画した際、または環境が変わった際は、直ちに **`docs/HARNESS.md`** に従って自分自身の検証環境（フック、リンター、検証スクリプト）をセットアップまたは確認してください。
