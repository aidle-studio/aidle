# Core Beliefs

本プロジェクトが大切にする設計思想と中核的な信念（Core Beliefs）を定義します。
AIエージェントは、実装において迷った際はこの信念に立ち返って判断してください。

## 🌟 中核的な信念

1. **AI-First / Agent-First**: 人間だけでなく、AIエージェントが「理解しやすく、自律的に開発を進められる」環境とコード構造を最優先する。
2. **Explicit over Implicit**: 暗黙の了解を排除し、すべてをリポジトリ内のドキュメントやコードとして明文化する。
3. **Failing Fast & Feedback Loop**: 検証結果を即座にAIへフィードバックし、自己修正可能なサイクル（Harness）を重視する。
4. **Minimalistic Implementation**: 常に最小限の実装で仕様を満たし、不要な複雑性を持ち込まない。

## 🛠️ 設計原理

- **不変性の尊重**: 状態を最小限にし、副作用の少ない、予測可能なコードを。
- **合成可能性**: 小さなコンポーネントや関数を組み合わせて、大きな機能を実現せよ。
- **型の安全性**: 型システムを最大限に活用し、コンパイル時点でバグを防げ。

## 🚦 行動規範

エージェントは、この信念と矛盾する実装や提案を避ける義務があります。
もし信念を更新する必要がある場合は、必ず人間との合意（Agreement Gate）を経て `core-beliefs.md` を更新してください。
