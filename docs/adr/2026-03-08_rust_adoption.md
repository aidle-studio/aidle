# ADR-0001: CLI実装言語としてRustを採用

- 日付: 2026-03-08
- ステータス: Accepted
- 決定者: プロジェクトチーム

### Context
- `aidle` はプロジェクト雛形を生成するCLIとして、OS間で安定して動作することが重要。
- 利用者の環境差異（ランタイム有無、バージョン不一致）による導入障壁を下げたい。
- 生成処理は多数のファイル作成を伴うため、起動速度と実行性能も重視する。

### Decision
- CLIの実装言語としてRustを採用する。

### Consequences
- メリット:
  - 単一バイナリ配布によってランタイム依存を減らせる。
  - 高速かつ安定したCLI実行が可能。
  - 型安全な実装で品質を担保しやすい。
- デメリット:
  - メンバーによっては学習コストが発生する。
  - ビルド時間が相対的に長くなる場合がある。

### Follow-ups
- 初期実装で使用するライブラリを確定する（`clap`, `serde`, テンプレートエンジン）。
- TDD方針に合わせ、CLI振る舞いの統合テスト方針を `TEST_PLAN.md` に反映する。
