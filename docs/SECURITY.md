# SECURITY.md

本ドキュメントは、プロジェクトのセキュリティ基準、機密情報の取り扱い、および安全なコーディングの指針を定義します。

## 🛡️ セキュリティ・ガードレール

AIエージェントは、以下のセキュリティルールを厳守してください。

1. **機密情報の保護**: APIキー、パスワード、トークン等を決してログ、画面、コミットに含めてはなりません。
2. **環境変数の利用**: 設定はソースコードに直書きせず、必ず環境変数または秘密情報管理ツールを経由してください。
3. **入力のバリデーション**: 外部からの入力は、常に汚染されている（Untrusted）ものとして扱い、厳密なバリデーションとサニタイズを実施してください。
4. **最小権限の原則 (PoLP)**: プログラムの実行権限、ファイルの読み書き権限などは、必要最小限に制限してください。

## 🔒 セキュリティ・スキャン

エージェントは、以下のセキュリティチェックをハーネス（検証レール）の一部として構築してください。

- **静的解析 (SAST)**: コード内の脆弱性を検知。
- **依存関係スキャン**: 既知の脆弱性があるパッケージの検知。
- **シークレットスキャン**: 誤ってコミットされた秘密情報の検知。

## 🚨 脆弱性発見時の対応

セキュリティ上の懸念を発見した場合は、速やかに `SECURITY.md` に追記し、人間へ報告してください。
公開されたIssueとして記録する前に、非公開の連絡ルートの有無を確認してください。
