# DESIGN.md

本ドキュメントは、プロジェクトのデザイン・UXガイドライン、共通コンポーネント、およびスタイリングの指針を定義します。

> **💡 ポインタ**
> ユーザー体験の根本的な考え方、ターゲット層のペルソナ、および「なぜこのデザインにするのか」という意図については、まず **[`PRODUCT_SENSE.md`](PRODUCT_SENSE.md)** を参照してください。本ドキュメントは、その意図を具体的なUIコンポーネントやスタイルに落とし込むための「実装指針」です。

## 🎨 デザインの原則

AIエージェントは、実装において以下のデザイン指針を遵守してください。

1. **一貫性**: 同じ機能、同じ操作には、同じデザインと挙動を。
2. **階層構造**: 情報の重要度を、タイポグラフィ、色、余白によって明確化せよ。
3. **インタラクティブ性**: ユーザーのアクションに対して、適切なフィードバックを。

## 🧩 共通コンポーネント

エージェントは、新規画面や機能の実装において、既存のコンポーネントを最大限に再利用してください。

- **Button**: 役割に応じた標準色（Primary, Secondary, Danger）を使用せよ。
- **Input**: バリデーションエラー時の表示を共通化せよ。
- **Layout**: グリッドシステムまたは標準的なレイアウトテンプレートに従え。

## 💄 スタイリング

- **CSS/UI Framework**: [例: Tailwind CSS / Shadcn UI] を標準とする。
- **命名規則**: [例: BEM / Utility-First] の方針に従え。
- **アクセシビリティ**: コントラスト比、スクリーンリーダー対応、キーボード操作性を確保せよ。
