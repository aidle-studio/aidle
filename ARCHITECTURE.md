# ARCHITECTURE.md

このファイルは、`aidle` プロジェクトのシステム設計・依存関係・階層構造を定義する **「不変の地図」** です。

## 🏛️ プロジェクトの論理構造

本プロジェクトは以下のディレクトリ階層で構成されています。

### / (Project Root)
- `AGENTS.md`: エージェント向けの極小ポインタ。
- `ARCHITECTURE.md`: 本ファイル。
- `Cargo.toml`: Rust プロジェクトの定義と依存関係。
- `docs/`: システム・オブ・レコード（全知識の集約）。
- `src/`: Rust ソースコード。
- `templates/`: 生成されるプロジェクトの雛形（ボイラープレート）。
- `tests/`: 統合テスト（Integration Tests）。

### /src (Implementation)
- `main.rs`: CLI のエントリーポイント、引数処理、および生成ロジックのコア。

### /templates (Blueprints)
- `default/`: 全てのプロジェクトの基盤となる「3 Pillars」構成のテンプレート。

## 🏗️ 依存関係のルール

- **単一バイナリ**: 外部ランタイムに依存せず、Rust の標準的なビルドプロセスで生成されたバイナリのみで動作する。
- **テンプレートの独立**: `src/` のロジックは `templates/` 配下のファイル構造に依存するが、テンプレート自体は特定のプログラミング言語に依存しない汎用的な構造を持つ。

## 📂 各ディレクトリの責任範囲

- `src/`: テンプレートの読み込み、パスの解決、ファイルの書き出し、ロールバック処理の実行。
- `templates/`: エージェントファーストな開発環境を実現するためのドキュメントとフォルダ構造の定義。
- `tests/`: CLI の振る舞い（終了コード、生成物、エラー契約）を `SPEC.md` に基づいて保証する。
