# SDD Workflows (仕様から実装への連携手順)

このドキュメントは、AIエージェントが `aidle` の「仕様駆動開発 (SDD)」の規約に従い、仕様からテスト、そして実装へと橋渡しをするためのプロフェッショナルな手順です。

## 1. 要求と受け入れ基準の定義 (in `SPEC.md` / `product-specs/`)
新しい機能や要件が発生した場合、すぐにコードを書かず、まずは仕様書を更新します。
- 新機能に対し `REQ-xxx` の ID を割り当てる。
- その要件が満たされたことを機械的に証明するための具体的な完了条件を `AC-xxx` としてリストアップする。

## 2. トレーサビリティの確保 (in `TEST_PLAN.md`)
仕様が定まったら、それを「実行可能な検証」に変換します。
- `TEST_PLAN.md` を開き、新しく定義した `AC-xxx` に対応するテストケース ID (`TC-xxx`) を作成する。
- マトリクス表に `TC ID | 対応 AC | 観点 | 初期状態(Red)` を追記する。
- 1つの `AC` に対して、正常系 (Happy path) だけでなく異常系 (Edge/Error) の `TC` も検討する。

## 3. テスト駆動開発 (TDD) の実行
- `TC-xxx` に基づき、失敗するテスト（Red）のコードを記述する。
- テストを実行し、意図通りに「失敗する」ことを確認する（Harnessによる確認）。
- テストを通すための最小限の実装（Green）を行う。
- リファクタリングを実施し、`scripts/check_harness.sh` が全て通過することを確認して完了 (DoD 達成) となる。
